### 技術選定（iOS 固定。理由と却下案を明記）
検証環境: Xcode 26.6 / Swift 6.3.3 / iOS SDK 26.5 / Simulator iOS 26.5（iPhone 17系）。

| 領域 | 採用 | 理由 | 却下した選択肢 |
|------|------|------|----------------|
| 言語 | **Swift 6（language mode 6・strict concurrency）** | 最新・並行安全をコンパイラが静的に担保。撮影は AVFoundation/CoreMotion の非同期が多く、データ競合をビルド時に潰したい | Swift 5 mode（並行安全の担保が弱い） |
| UI | **SwiftUI**（`@Observable`） | 宣言的で画面数の多い本アプリと相性良、iOS標準・依存ゼロ | UIKit（記述量が多い）、クロスPF（センサ同期でネイティブ優位・要件§1） |
| アーキテクチャ | **MVVM + プロトコル注入サービス** | カメラ/IMU/保存を protocol で抽象化し、実機ハード無しでロジックをテスト可能に | TCA等の重量FW（v1にはオーバースペック・外部依存増） |
| 永続化 | **ファイルベース（1投稿=1フォルダ: manifest.json + video.mov + imu.jsonl + annotation.m4a）** | 「原本保持」と将来のデータセット納品構造（LeRobotのepisode相当）に直結。actorでスレッド安全、テストは一時ディレクトリで容易 | SwiftData/CoreData（v1の局所データには過剰・strict concurrency下のテスト複雑化）。将来データ量増で再検討 |
| カメラ | **AVFoundation（AVCaptureSession/MovieFileOutput）** | 映像収録の標準。将来 ARKit の visual-inertial odometry へ差し替え可能に protocol 化 | ARKit を v1 から常用（重い。カメラ姿勢の常時推定は v1 の100件づくりには不要） |
| IMU | **CoreMotion（CMMotionManager deviceMotion）** | 加速度・ジャイロ・端末姿勢を固定サンプリングで取得。映像とホストクロック基準で時刻同期 | 生 accelerometer/gyro 単体（姿勢が取れない） |
| 時刻同期 | **収録開始のホスト時刻を基準に IMU を相対時刻で記録**、映像PTSと突合可能に | 要件「映像とIMUは常に時刻同期」の不変ルールを満たす最小実装 | 完全ハード同期（v1は後処理突合で十分） |
| アップロード | **v1では実サーバ送信を行わない**。ローカル保持＋エクスポート用manifest＋`SubmissionSyncService` protocol（既定はローカルのみ・sync=pending） | バックエンド未整備。かつ**ユーザーデータの外部送信は CLAUDE.md「止まって確認」の境界**。要件§5の「サーバ保持」は将来の同期実装として口だけ用意し、v1はローカル完結で誠実にスコープ管理 | v1から実アップロード（外部送信の境界に抵触・要確認事項） |
| テスト | **Swift Testing（`import Testing`）** を `xcodebuild test`（Simulator）で実行 | 最新の公式テストFW。ドメインロジック/VMを実機なしで検証 | XCTestのみ（新規なら Testing を採用） |
| 静的解析 | **SwiftLint 0.65** + **Swift6 strict concurrency + 警告をエラー化** | 二段の静的ゲート（スタイル/複雑度 + 並行安全/型） | コンパイラ警告のみ（スタイル/複雑度の検出が弱い） |
| プロジェクト生成 | **XcodeGen 2.45（`project.yml`）で生成、生成物`.xcodeproj`もコミットする方針** | 手書きpbxprojの事故回避＋再生成可能。コミットしてあれば clone 後は XcodeGen 無しでもビルド可（未コミット環境では `xcodegen generate`） | 手書きpbxproj（脆い）、tuist（重い） |

### M3 計画（着手前の分解）
撮影は本アプリの核。**重要な制約: iOS シミュレータにはカメラが無く、AVFoundation の実収録はシミュレータで動かない**（実機のみ）。よって「実機依存の収録」と「テスト可能な純粋ロジック」を protocol で分離し、テストは fake サービスで全フローを回す。

- **サービス抽象（protocol 注入）**:
  - `CameraCaptureService`: 収録開始/停止 → `RecordedVideo`（url/尺/解像度/鮮鋭度/明るさ）、ライブの明るさ/鮮鋭度サンプリング。実装は AVFoundation（実機）。
  - `MotionRecordingService`: 100Hz で `[IMUSample]` を収集（開始ホスト時刻基準の相対 t）。実装は CoreMotion。
  - fake 実装をテスト用に用意し、`CaptureViewModel` の全フロー（前チェック→収録→下書き生成→保存）を実機なしで検証。
- **純粋ロジック（テスト対象）**:
  - ポカヨケ**前チェック** `PrecaptureReadiness`: マウント装着・手元が画角（自己確認）＋明るさ・IMU/権限（計測）→ 開始可否。
  - **撮影中**: 経過時間・上限尺での**自動停止**（＝要件の「自動トリミング」を stop-at-max で実現し、後処理 export を避ける。M1 レビュー N-3 の整合をここで取る。評価器の上限超過チェックは安全網として残す）。ライブ警告（画角外れ/ブレ/暗さ）。
  - **撮影後**: `QualityEvaluator`（M1）で判定 → プレビュー・成否自己申告・任意音声アノテ（後付け）・**投稿前プライバシー確認ゲート**（`privacyConfirmed`）→ `SubmissionDraft` を組んで `SubmissionStore.save`。
- **時刻同期**: 収録開始のホスト時刻を基準に IMU の t を相対秒で記録（中核設計A）。
- **検証**: fake サービスで VM フローをユニットテスト＋既存の Store 保存検証。実機カメラ収録は device-only のため CI 対象外（devlog に明記）。ビルドは strict concurrency で AVFoundation/CoreMotion コードの整合を担保。

**判定**: 🔴 なし。コード上の完了条件（ビルド・テスト・静的解析 全緑／reviewer Blocker ゼロ／docs 実装一致／過去指摘の退行なし解消）を満たす。**残る唯一の DoD ギャップは F-1（実機動作確認）で、これは本環境では実施不能**。

## v1 最終サマリ（2026-07-12）

### 何を作ったか
gavel C側アプリ（供給側＝データ収集アプリ）v1 の iOS 実装。限定協力者が一人称マウント撮影で「映像＋IMU」を収集し端末内に原本保持する最小構成。規模: アプリ 45 ファイル/約3,400行、テスト 16 ファイル/約1,350行・**94件（Swift Testing）**。

- **オンボーディング**: welcome→同意（必須）→プロフィール（招待コード＋表示名＋マウント）→撮り方チュートリアル。
- **ミッション**: 日本特化の常設5件（箸盛り付け／和食器／狭い台所配膳／卵の扱い＝置き場所の判断／急須）。マーケットプレイスは持たない。
- **キャプチャ（ポカヨケ3段）**: 撮影前チェック（マウント/手元の自己確認＋明るさ/IMU計測）→ 収録（映像 AVFoundation＋IMU CoreMotion 100Hz、収録開始を共通起点に同期、上限60秒で自動停止、本編クリップは音声なし）→ レビュー（品質判定・成否自己申告・任意音声アノテ後付け・**プライバシー確認必須ゲート**）→ 保存。
- **保存**: 「1投稿=1フォルダ」（manifest.json＋video＋imu.jsonl＋任意 annotation）。copy→manifest コミット→元削除、失敗時ロールバック。
- **記録**: 100件ゴール進捗・ミッション別内訳・投稿履歴・詳細（再生＋メタ＋削除）。ウォレット無し（固定謝礼＋貢献記録）。

### 主要な判断（詳細は各追記）
- iOS 単独／Swift6 strict concurrency／SwiftUI `@Observable` MVVM／protocol 注入でハード非依存にテスト可能化。
- 永続化は SwiftData でなくファイルベース（原本保持＋将来のデータセット納品構造 LeRobot 相当に直結、テスト容易）。
- v1 は実サーバ送信を行わない（外部送信は要合意の境界・バックエンド未整備）。所有権は自社/供給者側に残す（`syncState=pendingSync`）。
- 検証は XcodeGen＋SwiftLint(--strict)＋xcodebuild test をワンコマンド化（`scripts/verify.sh`）。実機カメラ収録は device-only。
