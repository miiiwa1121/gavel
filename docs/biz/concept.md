# 事業コンセプト - gavel

> **要約**: このファイルは「gavel」（仮称）が**何をする事業か**を一言で示し、**三者構造・事業展開の3段階・差別化の軸・ポジショニング（何であり／何でないか）**をまとめた資料です。事業の全体像・方向性を最初に掴みたいときに読みます。
>
> 市場規模は [`market.md`](./market.md)、競合の詳細は [`competitors.md`](./competitors.md)、事業モデル（収益化）の詳細は [`business-model.md`](./business-model.md)、技術的な裏付けは [`tech-notes.md`](../tech/tech-notes.md) を参照。

## 事業概要

**一言で言うと**: 日本の一般消費者がスマホで日常の手元動作を撮影・投稿し、それを**フィジカルAI（身体性AI）・ロボット企業に「学習データの閲覧権（ライセンス）」として販売する、分散型データプラットフォーム事業**。

> "gavel" は仮のコードネーム。正式なサービス名は未定。

### 三者の構造

本事業は「売り手（供給側）」「プラットフォーム（自社）」「買い手（需要側）」の三者で成り立つ。**供給と需要を自社が仲介し、データの権利を保持したまま閲覧権を売る**のが骨格。

| 立場 | 役割 |
|------|------|
| **売り手（供給側）** | 日本国内の一般消費者。ポイ活・副業感覚でスマホ撮影し、ミッションに沿って投稿する。任意でアノテーション（音声実況）すると単価が上がる。**データが売れるたびに継続報酬（レベニューシェア）** を得る |
| **プラットフォーム（自社）** | ミッション配信、ポカヨケ撮影UIの提供、AI自動処理＋人力補完によるデータ加工、企業への**ライセンス販売・権利管理** |
| **買い手（需要側）** | Physical Intelligence、Figure、Boston Dynamics、国内ロボット企業（ugo 等）。**非独占ライセンス**（複数社に反復販売・低単価）と**独占ライセンス**（1社囲い込み・高単価）の二階建て |

三者の具体的な動き方（1日の使われ方・発注の流れ）は [`usecases.md`](./usecases.md) を参照。収益化メカニクスの詳細は [`business-model.md`](./business-model.md) を参照。

### 事業展開の3段階（初期構想）

1. **個人向け投稿プラットフォームの運営**（本丸。まずここで実績とデータ基盤を作る）
2. **ノウハウを活かした企業への導入支援事業**（収集の型を他社の内製化に提供）
3. **データ基盤を土台に、ロボットのソフトウェア領域そのものへ展開**

## 差別化の軸（3本柱）

1. **日本特化**: 箸使い、和食器、狭い台所での所作など、**海外勢が取りに来るコスパの悪いデータ**に的を絞る。
2. **暗黙知・判断の記録**: 単なる動作の軌跡ではなく「**なぜその判断をしたか**」を残す（例：卵をまな板ではなくコンロの隙間に置く理由）。この「判断・手順・暗黙知」は物理シミュレーションの外側にあり、現実データにしか宿らない（[`tech-notes.md`](../tech/tech-notes.md)）。
3. **ミッション形式**: 企業依頼に応じて集める**受注型**と、依頼がなくても日本特有データを先行確保する**常設型**の二本立て（[`business-model.md`](./business-model.md)）。

## ポジショニング — 何であり、何でないか

> 競合調査（[`competitors.md`](./competitors.md)）で、市場を「プロ収集 ↔ ToC分散」×「汎用 ↔ 日本特化」の2軸で切ると、**「ToC分散 × 日本特化」（右下）が空白地帯**であることが分かった。この立ち位置を判断のぶれ止めとして固定する。

| gavel は… | gavel では**ない** |
|-----------|---------------------|
| 一般消費者がスマホで集める **ToC分散型**のデータ収集 | プロ／契約ワーカー主導の受託収集（Scale AI・FastLabel 型） |
| **日本特化**（箸・和食器・狭い台所・判断/暗黙知）のニッチ在庫 | 汎用・グローバルの物量勝負（Asimov AI 型） |
| データの**閲覧権（ライセンス）を売る**権利管理事業 | データの所有権ごと売り切る単発の受託 |
| 特定レイヤー（環境・用途・日本特有）に特化 | 汎用「人間基盤モデル」を作るインフラ事業（NVIDIA 等が先行。[`business-model.md`](./business-model.md) の「見送ったアイデア」） |

### この立ち位置の前提（賭け・最重要の事業リスク）

- **空白は埋まりうる**: 日本勢2社が両側から迫っている。**FastLabel**（収集の裾野を OpenLUTRA で拡大）と **APTO/harBest**（汎用アノテーション→ロボット特化へシフト）。空白地帯は時間とともに埋まる可能性が高く、**実績づくりのスピードそのものが事業リスク**（[`roadmap.md`](./roadmap.md)）。
- **需給ギャップが構造的ボトルネック**: 市場分析で、供給側（データを集める力）は潤沢だが**需要側（買ってくれる企業の予算）が細い**ことが数値で判明した（トップダウンSOMとボトムアップSOMの乖離＝約10倍）。この乖離をどう事業設計で埋めるかが最大の論点（[`market.md`](./market.md)・[`roadmap.md`](./roadmap.md)）。

## なぜ今か（タイミング）

- LLM は「データ量に比例して性能が伸びる」脳筋プレイで進化してきたが、良質テキストの枯渇・収穫逓減で勢いに陰りが出ている。一方**フィジカルAI（ロボット）はまだ脳筋プレイにすら入れていない段階**で、学習データをこれから集めるフェーズにある。
- 「LLM は燃料切れで減速／ロボットは燃料をこれから集める」という**真逆のフェーズ**に本事業は立つ（詳細は [`industry.md`](./industry.md)）。
