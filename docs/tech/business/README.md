# tech/business — 需要側（買い手企業）プロダクト/技術【予約】

> **要約**: 本プラットフォームの**需要側＝データを買う企業（国内ロボット企業・海外フロンティアラボ）向けプラットフォーム**のプロダクト・技術設計を置く領域。現在は**企画・調査フェーズのため未使用**（プレースホルダ）。

## ここに置くもの（プロダクト設計に入ったら）

データを「買う」側＝企業がデータを発注・閲覧・購入する側の設計。

- 受注ミッションの発注・条件指定フロー
- データの検索・プレビュー・カタログ（常設在庫の閲覧）
- **ライセンス販売・権利管理**（非独占／独占の許諾、二重許諾の防止 → [`../data-model.md`](../data-model.md)）
- 納品形式（ロボット学習の標準フォーマット **LeRobot** 等 → [`../../biz/industry.md`](../../biz/industry.md)）
- 課金・契約、閲覧権の期限・範囲管理
