# tech/customer — 供給側（消費者）プロダクト/技術【予約】

> **要約**: 本プラットフォームの**供給側＝一般消費者向けデータ収集アプリ**のプロダクト・技術設計を置く領域。現在は**企画・調査フェーズのため未使用**（プレースホルダ）。

## ここに置くもの（プロダクト設計に入ったら）

データを「出す」側＝スマホで手元動作を撮影・投稿する消費者アプリに関する設計。

- 撮影・投稿フロー、**ポカヨケ撮影UI**（撮り方の失敗を構造的に防ぐ）
- ミッションの受け取り・一覧・進捗（受注型／常設型）
- 任意アノテーション（音声実況）の入力体験
- レベニューシェアの可視化（自分のデータが売れた／報酬の履歴）
- スマホのIMU＋映像の取得・端末側処理、器具配布時（簡易グリッパー等）の収集フロー
- アカウント・個人情報/プライバシー（顔ぼかし等）のクライアント側の扱い
