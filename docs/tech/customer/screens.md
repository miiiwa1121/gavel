# C側アプリ 画面一覧・画面遷移 - gavel

> **要約**: 供給側＝データ収集アプリ（C側）の画面と遷移の定義。要件（[`requirements.md`](./requirements.md)）を iOS アプリとして具体化したもの。**実装状況**列で M0〜M5 のどこで作るかを追う。モック（`docs/reference/datamine_prototype.jsx`）の画面骨格は参考にしつつ、v1 要件に無いもの（ウォレット/マーケットプレイス/写真モード）は採らない。

## 全体構成

起動時に保存済みプロフィール（`profile.json`）を読み、**同意済みなら本編（タブ）／未登録・未同意ならオンボーディング**へ分岐する（`SessionModel`）。

```
起動
 └─ RootView（フェーズ分岐）
     ├─ loading … 読み込み中
     ├─ onboarding … OnboardingView（同意していない/未登録）
     └─ main … MainTabView（同意済み）
          ├─ ホーム（HomeView）─────┐
          ├─ ミッション（MissionsView）┴→ ミッション詳細（MissionDetailView）→ 撮影（Capture）
          ├─ 記録（ContributionView）
          └─ 設定（SettingsView）
```

※ ミッション詳細へは**ホームからもミッション一覧からも**遷移できる（どちらも `Mission` を値に持つ `navigationDestination`）。

## 画面一覧

| 画面 | 役割 | 主な要素 | 実装状況 |
|------|------|----------|----------|
| **オンボーディング**（`OnboardingView`） | 同意取得と最初の教示 | welcome / 同意（`ConsentText`）/ プロフィール（表示名・招待コード・マウント）/ 撮り方チュートリアル | **M2 実装済** |
| **ホーム**（`HomeView`） | 起点・貢献の可視化 | あいさつ／貢献サマリカード／常設ミッション導線 | **M2 実装済** |
| **ミッション一覧**（`MissionsView`） | 常設ミッションの一覧 | 少数固定リスト（日本特化） | **M2 実装済** |
| **ミッション詳細**（`MissionDetailView`） | タスクの説明と撮影導線 | タスク説明／お手本／成功条件／撮り方の注意／撮影CTA | **M2 実装済** |
| **撮影**（`CaptureView`） | 映像＋IMU 同期収録・ポカヨケ第1-2段 | 撮影前チェック（マウント/手元の自己確認＋明るさ/IMU計測）／撮影中の経過時間・上限尺で**自動停止**。本編クリップは**映像＋IMUのみ（音声を含めない）** | **M3 実装済** |
| **レビュー/投稿**（`CaptureReviewView`） | ポカヨケ第3段・自己申告・任意音声・保存 | プレビュー／品質チェック結果（`QualityEvaluator`）／成否自己申告／音声アノテ（後付け・任意）／**プライバシー確認ゲート（必須）**／`SubmissionStore` へ保存 | **M3 実装済** |
| **記録**（`ContributionView`） | ウォレット代替の貢献記録 | 貢献サマリ／投稿履歴 | **M2 実装済**（履歴は M3 で実データが入る） |
| **設定**（`SettingsView`） | プロフィール・同意状態・アプリ情報 | 表示名／招待コード／マウント／同意日時／バージョン | **M2 実装済** |

## 主要遷移

- **撮影フロー（v1 の核）**: ミッション詳細の「このミッションで撮影する」→ 撮影（前チェック→収録→後確認）→ レビュー/投稿（自己申告・任意音声・プライバシー確認）→ 保存 → 記録に反映。**1投稿＝1タスクの1実演**（不変ルール）。
- **オンボーディング完了**: チュートリアル最終ステップで同意済みプロフィールを保存し、本編（タブ）へ。以降の起動はホーム直行。

## モックとの差分（意図的に採らないもの）

要件優先（①要件 > ②data-model > ③モック）に従い、モックにある以下は v1 では持たない: 案件マーケットプレイス（検索・企業名・締切・募集人数）、ウォレット/出金/KYC、写真モード、三人称撮影。撮影は独立タブでなく**ミッション詳細から開始**する（タスク単位の導線を崩さないため）。
