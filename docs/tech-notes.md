# 技術的論点 - gavel

> **要約**: 本事業の成否や差別化を左右する**技術的な論点**を整理した資料。「スマホ動画データはモーションキャプチャに勝てるか」「仮想データが進化しても現実データは要るか」「なぜ人型ロボットなのか」等、**プロダクト設計とポジショニングの前提になる技術的判断**を記録する。
>
> 事業の差別化との対応は [`concept.md`](./concept.md)、業界トレンド（スケーリング則等）は [`industry.md`](./industry.md)、出典は [`references.md`](./references.md) を参照。

## 1. 動画データ vs モーションキャプチャ（専用デバイス）

- **専用デバイス（UMI のグリッパー等）が担保するもの**: グリッパー開閉幅などの「**アクションラベル**」（動作の数値）。これは**スマホでは原理的に取得不可**。
- **スマホ単体でも取れるもの**: IMU（加速度・ジャイロ）と映像を組み合わせた **visual-inertial SLAM** により、UMI 並みの精度に近づける可能性（Fast-UMI 等の研究あり）。
- **UMI の仕様（参考）**: GoPro ＋ 155度魚眼レンズ ＋ ミラーで疑似ステレオ視、ORB-SLAM3 ベースの SLAM で **99% のデータがトラッキング成功**。
- **重要**: **Asimov AI はグリッパーでなくヘッドバンド方式**。本事業と同じ「専用精密器具なし」路線が、**最も資金調達している競合によって実証済み**（[`competitors.md`](./competitors.md)）。

## 2. データの「次元」の違い（活用方法の違い）

| データ種別 | AIに何を与えるか | 用途 |
|------------|------------------|------|
| **映像データ** | 「世界を理解させる」（概念・文脈・常識） | 事前学習（pre-training）向け |
| **モーションキャプチャ的データ** | 「体を動かすことを教える」（具体的な運動指令） | 微調整（fine-tuning）向け |
| **スマホ＋IMU** | 両者の**中間ゾーン** | 安価・大量・そこそこ動きの情報もある |

- 業界標準は「**シミュレーションで大量事前学習 → 少量の現実データで微調整**」のハイブリッド。
- **本事業のポジション**: スマホ＋IMU は「安価・大量・そこそこ動きあり」という中間を取れる。純粋なモーキャプ精度では専用デバイスに劣るが、**物量と単価で優位**に立てる。

## 3. 多様性と偽の相関（過学習の回避）

- 例:「卵をまな板に置かず、転がりにくい場所に置く」という判断行動。
- 対象物（卵）だけに反応する学習は「**偽の相関（spurious correlation）**」で誤り。
- **多様な物体（卵・リンゴ・ボール等）**で「転がる」という**本質**を学ばせる必要がある。
- **失敗データ**（まな板に置いて落ちた）にも「**なぜ失敗したか**」の文脈を付けることで、原因の取り違えを防げる。
- 言葉のラベルなしでも大量データからの推論（自己教師あり学習）は可能だが、**少量で正確に学ぶには言語アノテーションが有効**。「言葉なし＝大量でゴリ押し」「言葉あり＝少量で精密」というトレードオフ。
- → **示唆**: 本事業の「多様な環境・対象物」×「判断/失敗の言語アノテーション」（[`business-model.md`](./business-model.md)）は、この技術的要請に直接応える設計になっている。

## 4. 仮想データ（シミュレーター）vs 現実データ

- **2026年の業界標準**: 仮想データで大量事前学習 → **少量（50〜100件）の現実データで微調整**。
- **仮想データの強み**: 無限・低コスト・危険/レアケースの生成・プライバシー問題なし。3D Gaussian Splatting 等の技術進化で **sim-to-real ギャップは縮小中**（成功率 86% vs 現実データ 97.5% まで接近という報告あり）。
- **現実データが消えない理由**: 「**sim-to-real ギャップ**」（摩擦・変形・光・センサーノイズの完全再現は不可能）。タスクが複雑なほど現実データ比率が上がる。
- **将来、超高度な仮想データが実現しても**:
  1. 仮想データの"**種**"となる現実データは原理的に必要（現実を知らないと現実を再現できない）。
  2. 「**なぜその判断をしたか**」という人間の暗黙知は物理シミュレーションの外側にあり、**現実データにしか宿らない**。
- **結論**: 「凡庸な現実データ」の価値は下がりうるが、「**質の高い種データ**」と「**判断・暗黙知を含むデータ**」の価値は、仮想化が進むほど**相対的に上がる**。
- **発想の転換**: 仮想データ企業（NVIDIA 等）を**競合でなく顧客**（現実データの供給先）と捉える視点もあり得る（[`competitors.md`](./competitors.md)）。

## 5. 人型ロボットである必然性

- 特定タスクだけなら人型は非効率（**Mujin** は倉庫ピッキングに特化して成功）。
- **人型が支持される最大の理由**: 「**世界が人間用に設計されているから**」（既存環境を改造せず使える）。
- 次点の理由: 人間動作データを転用しやすい、汎用性への投資対効果、心理的受容性・資金調達上のインパクト。
- **本事業への示唆**: 人間動作データは**人型ロボット隆盛への賭け**に乗っている側面がある。ただし「**判断・手順・暗黙知**」は**非人型ロボットにも転用可能**であり、賭けが外れた場合の**リスクヘッジ**になる。
